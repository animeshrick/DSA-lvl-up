# 3-Month Interview Prep Roadmap — Service/IT Companies

**Candidate profile:** Intermediate DSA · Python · 2–3 hrs/day · Target: TCS, Infosys, Wipro, Accenture, Cognizant, Capgemini, etc.
**Start:** June 2026 · **Goal interview-ready:** September 2026

---

## How service companies actually select (read this first)

Your offer is decided across 4 gates, not just coding:

1. **Aptitude test** — Quantitative + Logical Reasoning + Verbal. This silently eliminates the most people. Treat it as seriously as coding.
2. **Coding round** — Usually 1–3 problems, **easy to lower-medium**. Heavy on patterns, number theory, strings, arrays. You rarely need graphs or advanced DP.
3. **Technical interview** — DSA basics *explained out loud* + DBMS + OOP + OS/Networking basics + **your resume projects**.
4. **HR round** — Communication, "why this company", stability, relocation.

**Takeaway:** DSA is necessary but not sufficient. Budget ~50% of time to DSA early, then rebalance toward aptitude + core CS + projects as the test nears.

## The 3-month arc

### Month 1 — Foundations (DSA Level-1 + Aptitude start)
- **DSA (60%):** Finish the full DSA Level-1 syllabus (see `01_DSA_Level1_syllabus.md`). Arrays, strings, math/number theory, patterns, searching/sorting, hashing, recursion basics.
- **Aptitude (30%):** Start quant fundamentals — percentages, ratios, averages, time-speed-distance, number system.
- **Core CS (10%):** Light reading — OOP 4 pillars, what is DBMS.
- **Milestone:** Comfortably solve any easy problem in Python in <15 min. Solve 100+ problems total.

### Month 2 — Consolidation (DSA Level-2 lite + Aptitude depth + Core CS)
- **DSA (40%):** Linked lists, stacks/queues, basic trees & BST traversals, two-pointer/sliding window, basic backtracking. Start timed coding (45 min, 2 problems).
- **Aptitude (30%):** Logical reasoning (series, coding-decoding, blood relations, puzzles) + verbal (RC, grammar, sentence correction).
- **Core CS (30%):** DBMS (SQL queries, normalization, joins, ACID), OOP (deep), OS basics (process/thread, deadlock, scheduling), CN basics (OSI, TCP/IP, HTTP).
- **Milestone:** Pass a full mock aptitude test (>70%). Explain your projects in 2 minutes each.

### Month 3 — Interview mode (Mocks + Revision + Behavioral)
- **DSA (30%):** Daily revision of weak topics + 2 timed problems/day. Pattern + string + math drills (most-asked in coding rounds).
- **Aptitude (20%):** Full-length timed mocks, 2–3 per week.
- **Core CS + Projects (30%):** Mock technical interviews. Prepare 8–10 SQL queries, common OS/DBMS Q&A, resume deep-dive.
- **HR (20%):** Prepare answers: introduce yourself, strengths/weaknesses, why this company, where in 5 years, conflict story. Mock HR rounds.
- **Milestone:** 3+ full mock interviews done. Resume polished. Confident on all 4 gates.

## Ground rules (recruiter's advice)

- **Consistency beats intensity.** 2 focused hours daily for 90 days >> weekend cramming.
- **Always dry-run before coding.** Interviewers watch *how* you think, not just the answer.
- **Talk while you solve.** Practice narrating your approach now, alone, so it's natural in interviews.
- **Revise, don't just collect.** A solved problem you can't redo in a month is not learned. Use the "Revise?" flag in the tracker.
- **Build a resume project story.** Even one solid project you can explain end-to-end beats five you can't.
- **Apply early.** Don't wait until "ready." Start applying in Month 2 to get real interview reps.
