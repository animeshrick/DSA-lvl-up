# Practice Problems — LeetCode & HackerRank

External practice to back up the daily curriculum. Every problem here maps to a topic you're
covering in the `DayXX` folders, so use them as extra reps once you've cleared a day's set.

## Files
- **`LeetCode_by_topic.md`** — ~40 problems (mostly Easy, a few Medium ⭐), grouped by topic:
  Numbers/Math, Arrays, Hashing, Strings, and a Binary Search / Two Pointers preview.
- **`HackerRank_by_topic.md`** — ~30 problems from the Interview Prep Kit, grouped the same way.
  HackerRank is what many service companies use for real online tests, so practice the STDIN/STDOUT
  parsing here.

## How to use
1. Finish the day's playground (all green) first.
2. Pick 2–3 problems from the matching topic section and solve them on the platform in Python.
3. Tick the checkbox `[x]` when accepted.
4. Flag anything that stumped you — we'll fold it into a revision day.

## Difficulty guide
For service/IT coding rounds, **Easy is the realistic target.** Mediums (⭐) are stretch reps
to build a buffer. Don't grind Hards yet — that's Level-2 territory.

## Platform study plans worth following
- LeetCode: *LeetCode 75* and *Top Interview 150*.
- HackerRank: *1 Week* and *3 Months Preparation Kit*.

*Note: platform URLs are stable problem slugs, but if any problem has been renamed/retired,
search the title on the platform — the slug almost always still resolves.*
